# Contributing to Beacon
We're always welcome to feedback and contributions from the community! However, there are good and bad ways to go about it.

## Issues
### Before making an issue
- Check if someone has already opened an issue for the problem you're experiencing
### To make a good issue:
- Explain the problem in detail
- If you can, tell us how to reproduce it
- Attach a picture of what happened if possible
- Tell us what time and date it happened
### Don't
- Share personal information in an issue
    - Issues are public and can be seen by anyone who accesses the repository, even without an account, so don't put anything personally identifiable in there!

 ## Contributing Code
 Have you fixed a bug or added a feature! Amazing! Add it by making a pull request from your fork.
 ### When Making PRs
 - Explain in detail what you've changed, and how you've done it
 - Be patient; we have a lot of things to look at, and of course we'll always consider your pull request when we get chance!

## Support
If you need help, you can:
- Ask in discussions here
- Send an email to [hello@beacondev.org](mailto:hello@beacondev.org)

Thank you very much for following these guidelines and helping to make Beacon better!
